Правила проекта

1 Порядок работы
1.1 Запрещено писать код сразу без обсуждения
1.2 По одному задаем все вопросы, которые могут прояснить задание
1.3 Получаем ответы
1.4 Предлагаем схему кода
1.5 Согласовываем структуру кода и функционал
1.6 После согласования начинаем писать код
1.7 Комментарии и документация - только на английском языке
1.8 Тупые комментарии (когда и так ясно, что делает код) не пишем
1.9 При исправлении ошибок запрещено ставить заплатки в код, без выяснения причины ошибки

2. Структура кода
2.1 Шаблоны
2.1.1 в HTML только шаблон
2.1.2 прямая стилизация в шаблоне (style) запрещена 
2.1.3 в шаблон не записываются скрипты
2.1.4 в шаблон не записывается описание классов
2.1.5 возможно использование нескольких скриптов, если они отвечают за поведение изолированных частей шаблона

2.2 Стили
2.2.1 Стилизация только через классы
2.2.2 Все файлы стилей находятся в папке css/scss
2.2.3 общий файл стилей style.scss находится в папке css 
2.2.4 Классы, на которые ориентируется JavaScript не имеют стилей
2.2.5 Для цветов используются _vars.scss	
2.2.6 Основные стилизации берем из _main.scss, его редактировать запрещено
2.2.7 Наиболее часто встречающиеся наборы стилей описываем в _mixins.scss
2.2.8 классы в отдельном файле _*.scss, который относится к данному шаблону
2.2.9 названия классов - по технологии БЭМ c использованием нижних подчеркиваний
2.2.10 Запрещено использование стилей без класса (по принципу .class div{}) то есть нельзя стилизовать дочерние элементы через класс родительского
2.2.11 Для цветов используются _vars.scss
2.2.12 Различные анимации - только по согласованию
2.2.13 Неиспользуемые после смены кода классы немедленно удаляются
2.2.14 Новые свойства, к существующему классу добавляются через дополнительный класс
2.2.15 Адаптив @media записывается строго после описания всех классов
2.2.16 Порядок классов в адаптиве такой же как в основном описании классов
2.2.17 В стилизации элементов опираемся на _main, _mixins

2.3 JavaScript
2.3.1 основной скрипт (только запуск функционала) в папке js 
2.3.2 модули в папке js/имя шаблона
2.3.3 допускается создание папки js/имя_шаблона/имя_модуля если модуль использует много других модулей, которые используются только им
2.3.3 в каждом модуле только одна функция
2.3.4 каждый модуль выполняет только одну операцию и вызывается во всех местах кода, где требуется эта операция
2.3.5 ни один модуль не может дублировать операцию другого модуля
2.3.6 код модуля не может повторять более чем на 25% код другого модуля
2.3.7 Названия модулей == название функций в нем
2.3.8 Названия функций должны однозначно отражать то, что они делают (согласовываются и предлагается несколько вариантов на выбор с описанием того, что делает модуль)
2.3.9 Используем Canvas
2.3.10 должна быть максимальная изоляция модулей, чтобы изменения в одном никак не отражались на работоспособности другого
